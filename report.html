Website Report

This report explains the design, development, validation, and responsive features of my portfolio website. The website was created as part of the Web Development module and demonstrates my understanding of HTML, CSS styling, responsive layout design, structure, and the process of building a multi-page portfolio website.

Design Choices

When designing my website, I wanted to create a clear and professional layout that would be easy to navigate. I used a simple colour scheme with blue tones, white content sections, and consistent spacing across all pages. This helped to create a clean visual style while keeping the website readable and organised. I also used semantic HTML elements such as header, nav, main, section, article, and footer so that the structure of the website is clear and meaningful.

Development Process

I developed the website in stages. First, I created the required HTML pages and linked them together using a shared navigation menu. After that, I added a common stylesheet so that the pages had a consistent design. I then styled the header, navigation, body, and content sections before creating a different grid layout for the Projects page. The project page was designed differently from the other pages so that it could better showcase project cards using CSS Grid.

Responsive Design

To make the website responsive, I used CSS media queries so that the layout could adapt to smaller screen sizes. For example, the project grid changes from two columns on larger screens to one column on smaller screens. I also adjusted the navigation links so that they stack vertically on smaller devices. This helped to improve readability and usability on mobile screens.

Challenges and Reflection

Although this was not my first time using HTML and CSS, this project was still valuable because it helped strengthen my understanding of the fundamentals. Before this module, I had already created different web pages, websites, dashboards, and applications, but I often felt that my knowledge had developed mainly through practice without always having a strong foundation. This assignment helped me focus more carefully on core principles such as semantic HTML structure, consistent CSS styling, responsive layout design, and organising code in a clearer, more maintainable way. It also showed me the importance of building websites in a structured way rather than focusing only on the final appearance. Another important part of the project was using GitHub, which helped me understand the value of version control, meaningful commits, and managing development more professionally over time. If I were to develop this website further, I would add more real project content, improve the visual design, and introduce simple interactive features using JavaScript.
